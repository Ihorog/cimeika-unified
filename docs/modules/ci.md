# Ci — Центральне ядро

## 1. Призначення

**Ci** (Сімейка) — це центральний модуль оркестрації системи Cimeika. Він координує роботу всіх інших модулів, забезпечує комунікацію між ними та управляє глобальним станом додатку.

### Ключові функції:
- Оркестрація взаємодії між модулями
- Управління глобальним контекстом
- Координація потоків даних
- Централізована обробка подій
- Моніторинг стану системи

## 2. UI-секції

### Frontend компоненти:
- **CiView** (`/ci`) — головний екран модуля
- **CiFAB** — глобальна кнопка доступу до Ci (присутня на всіх екранах)
- **Ci Overlay** — панель швидкого доступу з навігацією та командами

### Основні UI елементи:
1. **Навігація модулів** — швидкий перехід до всіх 7 модулів системи
2. **Пошук** — пошук модулів та команд
3. **Швидкі дії** — створення записів, статистика, налаштування
4. **Статус системи** — відображення здоров'я системи та версії

## 3. API

### Endpoints:

#### `GET /api/ci/state`
Отримати поточний стан модуля Ci
```json
{
  "status": "ready",
  "modules": [...],
  "health": {...}
}
```

#### `POST /api/ci/chat`
Надіслати повідомлення до Ci
```json
Request:
{
  "message": "текст повідомлення",
  "context": {}
}

Response:
{
  "reply": "відповідь від Ci",
  "timestamp": "2024-01-01T00:00:00Z",
  "context": {}
}
```

#### `GET /health`
Перевірка здоров'я системи
```json
{
  "status": "healthy",
  "message": "Backend is running",
  "version": "0.1.0",
  "timestamp": "2024-01-01T00:00:00Z"
}
```

## 4. Стани

### Можливі стани модуля:
- **idle** — очікування
- **loading** — завантаження даних
- **ready** — готовий до роботи
- **error** — помилка

### Обробка станів:
- **loading** → показати індикатор завантаження
- **empty** → показати повідомлення "Дані відсутні"
- **error** → показати повідомлення про помилку
- **ready** → відобразити дані

## 5. Зв'язки з модулями

Ci взаємодіє з усіма модулями системи:

- **PoDija** — координація подій та сценаріїв
- **Nastrij** — моніторинг емоційного стану
- **Malya** — управління ідеями та інноваціями
- **Kazkar** — доступ до історичних даних
- **Calendar** — синхронізація часових подій
- **Gallery** — управління медіа-контентом

### Принцип взаємодії:
- Ci НЕ містить доменної логіки інших модулів
- Взаємодія через інтерфейси та API
- Координація через події та глобальний стан
