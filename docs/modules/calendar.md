# Calendar — Календар

## 1. Призначення

**Calendar** (Календар) — модуль для управління часом, ритмами життя та планування. Організація подій, завдань та нагадувань у єдиному календарному просторі.

### Ключові функції:
- Планування подій та завдань
- Відстеження життєвих ритмів
- Інтеграція з модулем ПоДія
- Нагадування та сповіщення
- Різні види календарів
- Аналітика використання часу

## 2. UI-секції

### Frontend компоненти:
- **CalendarView** (`/calendar`) — головний екран календаря
- **EventCard** — картка події
- Календарна сітка
- Список подій
- Форма створення події

### Основні UI елементи:
1. **Огляд календаря** — місячний/тижневий/денний вигляд
2. **Список подій** — перелік майбутніх подій
3. **Форма події** — створення/редагування подій
4. **Фільтри** — за типом, статусом, тегами

## 3. API

### Endpoints:

#### `GET /api/calendar/events`
Отримати всі події календаря
```json
{
  "events": [...],
  "count": 10
}
```

#### `POST /api/calendar/events`
Створити нову подію
```json
Request:
{
  "title": "Назва події",
  "description": "Опис",
  "start": "2024-01-01T10:00:00Z",
  "end": "2024-01-01T11:00:00Z",
  "type": "event",
  "tags": ["робота"]
}

Response: CalendarEvent object
```

#### `GET /api/calendar/events/{id}`
Отримати подію за ID

#### `PUT /api/calendar/events/{id}`
Оновити подію

#### `DELETE /api/calendar/events/{id}`
Видалити подію

## 4. Стани

### Можливі стані:
- **idle** — очікування
- **loading** — завантаження подій
- **ready** — події завантажені
- **error** — помилка завантаження

### Типи подій:
- **event** — звичайна подія
- **task** — завдання
- **reminder** — нагадування

### Статуси подій:
- **pending** — очікується
- **completed** — виконано
- **cancelled** — скасовано

## 5. Зв'язки з модулями

### Інтеграція з іншими модулями:
- **PoDija** — синхронізація майбутніх сценаріїв
- **Nastrij** — планування на основі емоційного стану
- **Ci** — централізована координація часу
- **Malya** — планування реалізації ідей
